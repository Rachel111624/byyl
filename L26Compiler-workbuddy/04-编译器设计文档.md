## 1. L26 语言扩展后完整文法定义（Coco/R ATG 格式）

> 【原生文法】= PDF 原文定义；【扩展优化】= 本实现新增

```
COMPILER L26

CHARACTERS
    letter  = 'A'..'Z' + 'a'..'z' + '_'.
    digit   = '0'..'9'.
    cr      = '\r'.
    lf      = '\n'.
    tab     = '\t'.

TOKENS
    ident   = letter { letter | digit }.
    number  = digit  { digit }.

COMMENTS FROM "//"  TO lf              /* 【扩展优化】单行注释 */
COMMENTS FROM "/*"  TO "*/" NESTED     /* 【扩展优化】块注释，支持嵌套 */

IGNORE cr + lf + tab + ' '

PRODUCTIONS

/* -------- 顶层 -------- */
L26 = Block .                           /* 【原生文法】<program> ::= <block> */

/* -------- 块（作用域）-------- */
Block = "{" Decls Stmts "}" .           /* 【原生文法】<block> ::= "{" <decls> <stmts> "}" */

/* -------- 声明 -------- */
Decls = { Decl } .                     /* 【原生文法】<decls> ::= { <decl> }               */
Decl  = Type ident ";" .               /* 【原生文法】<decl> ::= <type> <ID> ";"            */
Type  = "int" | "bool" | "set" .       /* 【原生文法】<type> ::= "int"|"bool"|"set"         */

/* -------- 语句序列 -------- */
Stmts = { Stmt } .                     /* 【原生文法】<stmts> ::= { <stmt> }                */

Stmt  = AssignStmt                     /* 【原生文法】<stmt> 各分支                          */
      | IfStmt
      | WhileStmt
      | IOStmt
      | Block
      | SetOpStmt
      .

/* -------- 赋值 -------- */
AssignStmt = ident "=" Expr ";" .      /* 【原生文法】<assign_stmt>                          */

/* -------- if/while -------- */
IfStmt    = "if" "(" BExpr ")" Stmt
              [ "else" Stmt ] .        /* 【原生文法】<if_stmt>                               */
WhileStmt = "while" "(" BExpr ")" Stmt . /* 【原生文法】<while_stmt>                         */

/* -------- I/O -------- */
IOStmt = "write" Expr ";"
       | "read"  ident ";" .           /* 【原生文法】<io_stmt>                               */

/* -------- 集合操作语句 -------- */
SetOpStmt = "add"    ident AExpr ";"   /* 【原生文法】向集合添加元素                           */
          | "remove" ident AExpr ";" . /* 【原生文法】从集合删除元素                           */

/* -------- 通用表达式（前置谓词 + 后置检查）-------- */
Expr = IF (IsSetExprStart()) SetExpr
     | IF (IsAExprStart()) AExpr [ relop AExpr | "in" ident | "union" ident | "inter" ident ]
     | BExpr
     .

/* -------- 算术表达式 -------- */
AExpr  = ATerm { ("+" | "-") ATerm } . /* 【原生文法】<aexpr>                                */
ATerm  = AFactor { ("*" | "/") AFactor } . /* 【原生文法】<aterm>                            */
AFactor = ident | number | "-" AFactor | "(" AExpr ")" . /* 【原生文法】<afactor>            */

/* -------- 逻辑表达式 -------- */
BExpr   = BTerm { "||" BTerm } .       /* 【原生文法】<bexpr>                                 */
BTerm   = BFactor { "&&" BFactor } .   /* 【原生文法】<bterm>                                 */

/* -------- 布尔因子（Rel + SetTest + 单独bool变量 合并）-------- */
BFactor = "true"                       /* 【原生文法】<bfactor>                               */
        | "false"
        | "!" BFactor
        | "(" BExpr ")"
        | "isempty" "(" ident ")"
        | AExpr [ "in" ident | relop AExpr ]
        .

/* -------- 集合表达式 -------- */
SetExpr = "{" [ AExpr { "," AExpr } ] "}"  /* 【原生文法】集合字面量                          */
        | ident "union" ident               /* 【原生文法】并集                                */
        | ident "inter" ident .             /* 【原生文法】交集                                */

END L26.
```

**扩展语义说明（超出 PDF 文法，本实现新增）**：

| 扩展功能 | 说明 |
|----------|------|
| `//` 单行注释 | ATG `COMMENTS FROM "//" TO lf` |
| `/* */` 块注释（可嵌套） | ATG `COMMENTS FROM "/*" TO "*/" NESTED` |
| 集合相等 `SET_EQ` | `==` 操作数类型为 set 时自动发射 `OPR 0 208` |
| 集合不等 `SET_NE` | `!=` 操作数类型为 set 时自动发射 `OPR 0 211` |
| 变量名语义遮蔽 | 嵌套作用域 Lookup 向上查找 |
| BFactor 重构 | `Rel` + `SetTest` + 单独 bool 变量统一为 `AExpr [后置检查]` |

## 2. 编译器整体设计思路

本编译器采用 **Coco/R 驱动** 的单遍编译策略，在语法分析的同时完成语义分析和代码生成，无须显式构建 AST：

```
L26 源程序
    │
    ▼
Coco/R Scanner（自动生成）
    │ Token 流
    ▼
Coco/R Parser（ATG + 语义动作）
    ├── 调用 SymTab：声明/查找变量，检查类型
    └── 调用 PCodeGen.Emit*() ：生成 P-Code 指令
    │
    ▼
P-Code 指令序列（vector<PInstr>）
    │
    ▼
VM（类 P-Code 虚拟机）
    │ 逐条解释执行
    ▼
程序输出
```

**选择单遍编译的理由**：
- L26 语言不含函数/过程定义，无前向引用问题
- Coco/R 的 LL(k) 框架天然支持在产生式规则中嵌入任意 C++ 代码
- 符合参考项目 ExpressionProject 的 PL/0 实现惯例

## 3. Coco/R 扫描器与 Parser 工作流程

### 3.1 扫描器（Scanner.cpp）

Coco/R 根据 ATG 文件的 `CHARACTERS` 和 `TOKENS` 节，自动生成确定性有限自动机（DFA）：

```
CHARACTERS 节 → 字符集定义（letter, digit, ...）
TOKENS 节     → Token 识别规则
    ident  = letter { letter | digit } .   → 标识符 / 关键字
    number = digit  { digit } .             → 整数字面量

COMMENTS 节   → 跳过 // 和 /* */ 注释
IGNORE 节     → 跳过空白字符
```

生成的 `Scanner::Scan()` 方法每次调用返回下一个 `Token`（kind, val, line, col）。

关键字（`int bool set if else while write read add remove true false in isempty union inter`）由 `KeywordMap` 在扫描阶段自动识别，不需要在 ATG `TOKENS` 里逐一列举。

### 3.2 解析器（Parser.cpp）

Coco/R 将 `PRODUCTIONS` 节翻译为递归下降的 LL(1) 解析器：

- 每个产生式 → 一个 `Parser::ProductionName()` 方法
- `la`：当前前看 Token（lookahead），`la->next` 在 Coco/R C++ 运行时始终为 NULL
- `t`：上一个已消耗 Token
- `Get()` ：消耗一个 Token
- `Expect(n)` ：断言下一个 Token 是 n，否则调用 `SynErr()` 报错
- `StartOf(s)` ：用于处理 EBNF 的 `{` 和 `[` 重复/可选
- `IF(cond)` Coco/R 扩展：根据语义谓词选择产生式分支（解决 `Expr` 的歧义）

语义动作 `(. ... .)` 中的 C++ 代码直接嵌入生成的方法体内，在每个语法规则被匹配时执行。

### 3.3 Token 编号获取

**Coco/R 的 token 编号不可预测**，必须从生成的 `Parser.cpp` 注释中提取，例如：

```cpp
case 28 /* "true" */: ...
if (la->kind == 36 /* "union" */) ...
```

谓词函数 `IsSetExprStart()`、`IsAExprStart()`、`IsBExprOnly()`、`IsSetTest()` 中使用的 token 编号必须与 `Parser.cpp` 生成的编号完全一致，否则谓词判断错误，导致语法分析失败。

## 4. 符号表与语义检查设计

### 4.1 符号表结构

符号表使用**栈式多级作用域**实现（对应 L26 的嵌套 `<block>`）：

```
每个 Block 对应一个作用域层：
  EnterScope() → scopes.push_back(空 map)
  LeaveScope() → scopes.pop_back()

Lookup(name)：从栈顶向下逐层查找
Declare(name, type)：在当前层（栈顶）声明
```

**变量遮蔽语义**（PDF 示例2）：内层 `set x` 会遮蔽外层 `int x`；离开内层作用域后，`Lookup("x")` 重新找到外层的 `int x`。

### 4.2 语义检查项目

| 检查项 | 实现位置 | 错误处理 |
|--------|----------|----------|
| 变量重复声明 | `Decl` 语义动作 | `SemError("变量重复声明")` |
| 使用未声明变量 | `AFactor` / `AssignStmt` 语义动作 | `SemError("未声明的变量")` |
| 赋值类型检查 | `AssignStmt` 语义动作 | `SemError("赋值类型不匹配")` |
| `add`/`remove` 目标必须是 set | `SetOpStmt` 语义动作 | `SemError("add 目标必须是 set 类型")` |
| `in` 右侧必须是 set | `Expr`/`BFactor` 语义动作 | `SemError("in 右侧必须是 set")` |
| `union`/`inter` 操作数必须是 set | `Expr`/`SetExpr` 语义动作 | `SemError("union/inter 侧必须是 set")` |
| 集合不支持的关系运算符（<，>，<=，>=） | `Expr`/`BFactor` 语义动作 | `SemError("集合只支持 == 与 !=")` |

## 5. P-Code 中间代码设计

### 5.1 指令格式（标准 PL/0 三字段）

```c
struct PInstr {
    fct  f;    /* 指令功能码（枚举：LIT/OPR/LOD/STO/CAL/INT/JMP/JPC） */
    int  l;    /* 层次差（静态链向上跳几层；LIT/OPR/INT/JMP/JPC 时为 0）*/
    int  a;    /* 操作数（常量值 / 变量偏移 / 跳转地址 / OPR 子码）     */
};
```

> 参考：Pcode.md §一 《P-Code 指令格式》

### 5.2 完整指令集

#### 5.2.1 八种基本指令

| 指令 | 格式 | 语义 |
|------|------|------|
| `LIT` | `LIT 0 a` | 将常数 a 压入栈顶（s[++t] = a） |
| `OPR` | `OPR 0 x` | 执行子码 x 对应的运算（见下表） |
| `LOD` | `LOD l a` | 将 base(l, b) + a 处的变量值压栈 |
| `STO` | `STO l a` | 弹出栈顶，存入 base(l, b) + a 处 |
| `CAL` | `CAL l a` | 调用静态链向上 l 层的过程（地址 a） |
| `INT` | `INT 0 a` | 栈顶指针上移 a（为活动记录分配空间） |
| `JMP` | `JMP 0 a` | 无条件跳转：p = a |
| `JPC` | `JPC 0 a` | 弹出栈顶，若为 0（假）则 p = a |

#### 5.2.2 OPR 子码表（`OPR 0 x`）

** 标准 PL/0 子码（x = 0 … 17）**

| x | 操作 | 说明 |
|---|------|------|
| 0 | RET | 过程/程序返回：t=b-1, p=s[b+2], b=s[b+1] |
| 1 | NEG | 取负：s[t] = -s[t] |
| 2 | ADD | 加：t--; s[t] = s[t] + s[t+1] |
| 3 | SUB | 减：t--; s[t] = s[t] - s[t+1] |
| 4 | MUL | 乘：t--; s[t] = s[t] * s[t+1] |
| 5 | DIV | 除：t--; s[t] = s[t] / s[t+1] |
| 6 | MOD | 模：t--; s[t] = s[t] % s[t+1] |
| 7 | ODD | 奇数测试：s[t] = s[t] & 1 |
| 8 | EQL | 相等：t--; s[t] = (s[t] == s[t+1]) |
| 9 | NEQ | 不等：t--; s[t] = (s[t] != s[t+1]) |
| 10 | LSS | 小于：t--; s[t] = (s[t] < s[t+1]) |
| 11 | LEQ | 小于等于：t--; s[t] = (s[t] <= s[t+1]) |
| 12 | GTR | 大于：t--; s[t] = (s[t] > s[t+1]) |
| 13 | GEQ | 大于等于：t--; s[t] = (s[t] >= s[t+1]) |
| 14 | WRIT_BOOL | 弹出布尔，输出 true/false |
| 15 | WRIT_SET | 弹出集合，输出 `{a, b, c}` |
| 16 | READ | 读输入压栈（用于 `read` 语句） |
| 17 | WRIT | 弹出整数，输出（用于 `write` 语句） |

**L26 扩展子码（x ≥ 18）【扩展优化】**

| x | 操作 | 说明 |
|---|------|------|
| 18 | AND | 逻辑与：t--; s[t] = s[t] && s[t+1] |
| 19 | OR | 逻辑或：t--; s[t] = s[t] \|\| s[t+1] |
| 20 | NOT | 逻辑非：s[t] = !s[t] |
| 21 | LT | 小于比较 |
| 22 | LE | 小于等于比较 |
| 23 | GT | 大于比较 |
| 24 | GE | 大于等于比较 |
| 25 | NE | 不等于比较 |
| 201 | SET_MK | 从栈顶弹 a 个整数构造集合，结果存 g_setPool[t] |
| 202 | SET_ADD | 弹出整数，加入栈顶集合（保唯一性） |
| 203 | SET_REM | 弹出整数，从栈顶集合删除（不存在则不变） |
| 204 | SET_UNI | 弹出两集合求并集，结果压栈 |
| 205 | SET_INT | 弹出两集合求交集，结果压栈 |
| 206 | SET_IN | 弹出整数和集合，测试成员关系，结果压 0/1 |
| 207 | SET_EMP | 测试栈顶集合是否为空，结果压 0/1 |
| 208 | SET_EQ | 测试两集合是否相等，结果压 0/1【加分项】 |
| 211 | SET_NE | 测试两集合是否不等，结果压 0/1【加分项】 |

### 5.3 活动记录布局

```
s[b+0] = SL（Static Link，外层过程的基址 b）
s[b+1] = DL（Dynamic Link，调用方的 b）
s[b+2] = RA（Return Address，调用方的 p）
s[b+3] = 本地变量 0（第 1 个声明的变量）
s[b+4] = 本地变量 1
...
s[b+2+n] = 本地变量 n-1
```

> `INT 0 (n+3)` 为 n 个本地变量分配活动记录（n + 3 个槽）。
> `LOD l a` / `STO l a` 的偏移 a 从 3 开始（a = offset + 3）。

静态链追溯函数：
```c
int Base(int l, int b) {
    while (l > 0) { b = s[b]; l--; }
    return b;
}
/* LOD l a : s[++t] = s[Base(l,b) + a]; */
/* STO l a : s[Base(l,b) + a] = s[t--]; */
```

### 5.4 代码生成示例

**示例：`s = {1, 2, 3};`**

```
(addr)  f    l    a        ; 注释
  (k)  LIT   0    1        ; 压 1
 (k+1) LIT   0    2        ; 压 2
 (k+2) LIT   0    3        ; 压 3
 (k+3) OPR   0  201        ; SET_MK：弹3个构造集合
 (k+4) STO   0    4        ; STO 0 (offset_s+3)，存入 s
```

**示例：`if (val in s) { add s 10; }`**

```
(addr)  f    l    a        ; 注释
  (k)  LOD   0    3        ; LOD 0 (offset_val+3)，val 压栈
 (k+1) LOD   0    4        ; LOD 0 (offset_s+3)，s 压栈
 (k+2) OPR   0  206        ; SET_IN，弹两值，结果 0/1 压栈
 (k+3) JPC   0   ?         ; 假则跳过 if 体（回填）
 (k+4) LIT   0   10        ; 压 10
 (k+5) LOD   0    4        ; 取 s
 (k+6) OPR   0  202        ; SET_ADD
 (k+7) STO   0    4        ; 写回 s
 (k+8)         ← JPC 回填目标
```

**示例：`while (val > 0) { val = val - 1; write val; }`**

```
(addr)  f    l    a
  (k)  LOD   0    3        ; 加载 val（while 顶部，labelTop = k）
 (k+1) LIT   0    0        ; 压 0
 (k+2) OPR   0   23        ; GT（>）
 (k+3) JPC   0   ?         ; 假则跳出循环（回填 labelEnd）
 (k+4) LOD   0    3        ; val
 (k+5) LIT   0    1
 (k+6) OPR   0    3        ; SUB
 (k+7) STO   0    3        ; val = val - 1
 (k+8) LOD   0    3        ; write val
 (k+9) OPR   0   17
 (k+10) JMP  0    k        ; 跳回 labelTop
 (k+11)         ← JPC 回填目标（labelEnd）
```

**示例：`r = (s == t);`（s 和 t 均为 set 类型）**

```
(addr)  f    l    a
  (k)  LOD   0    3        ; 加载 s（set pool index）
 (k+1) LOD   0    4        ; 加载 t
 (k+2) OPR   0  208        ; SET_EQ → VM 内用 C++ std::set::operator== 逐元素比较
 (k+3) STO   0    5        ; 存入 r
```

main.cpp

## 附录A：第三组加分项实现情况

## 附录B：文法二义性处理说明

L26 的 `Expr` 有三个分支（`BExpr`, `AExpr`, `SetExpr`），存在 LL 歧义。本实现采用**前置谓词 + 后置检查**混合策略：

### B.1 前置谓词

用于仅能被特定产生式匹配的 token（不会被 `AExpr` 匹配到）：

- `IsSetExprStart()` ：检查 `la->kind == 3`（`{`）或 `ident` + `union`/`inter`（通过 `la->next`）
- `IsAExprStart()` ：检查 `la->kind` 为 `_ident` / `_number` / `"("` / `"-"`，并排除 `la->next` 使表达式变为 bool/set 的情况

### B.2 后置检查

对于 `ident` 的歧义（`s union t` / `x in s` / `x > 0`），先调用 `AExpr` 消费 ident，再用 `[ ... ]` 检查下一个 token：

- `[ relop AExpr ]` → 关系表达式（返回 bool）
- `[ "in" ident ]` → 集合成员测试（返回 bool）
- `[ "union" ident | "inter" ident ]` → 集合运算（返回 set）

### B.3 关键发现：`la->next` 陷阱

Coco/R C++ 版运行时 `la->next` **始终为 NULL**，所有依赖 2-token 前瞻的谓词（如检测 `ident + union/inter`）必须改用后置检查策略。

### B.4 BFactor 重构

原始文法的 `Rel`、`SetTest` 产生式已删除，逻辑完全合并到 `BFactor` 的 `AExpr [后置检查]` 中。这消除了 Coco/R 的 "Resolver will never be evaluated" 警告，同时支持了单独的 bool 变量（如 `if (b)`）的解析。
