# 1 项目概述

## 1.1 作业需求说明
根据《2026编译实践大作业要求》：
1. 实现扩展L26语言编译器，支持`int`整型、`bool`布尔、`set`集合三种数据类型；
2. set集合元素上限200；
3. 编译器输出标准类P-Code三字段指令，配套虚拟机解释运行；
4. 语言语法包含块、变量声明、赋值、if-else、while循环、读写语句、集合专用语法；
5. 集合功能：集合字面量`{}`、`union`并集、`inter`交集、`add/remove`增删、`in`成员判断、`isempty`空集判断；
6. 支持多层嵌套作用域、变量遮蔽、完整静态语义检查；
7. 界面展示P-Code清单、内存管理、单步运行（加分项）；
8. 集合相等`==`/不等`!=`判定（加分项）。

## 1.2 开发方案选型
本项目选择**第三组 Coco/R C++实现**：
- Coco/R从ATG文法文件自动生成词法扫描器与LL(1)递归下降语法解析器；
- 在ATG文法中嵌入语义动作，同步完成符号表管理、类型检查、P-Code代码生成；
- 采用标准PL/0三字段P-Code格式，设计集合扩展OPR子码；
- 前端提供Web可视化界面，通过Node.js桥接调用C++后端。

## 1.3 整体架构

```
L26源程序(.l26)
    │
    ▼
Coco/R Scanner（自动生成）→ Token流
    │
    ▼
Coco/R Parser（ATG + 语义动作）
    ├── SymTab：嵌套作用域管理、类型检查
    └── PCodeGen：生成标准三字段P-Code指令
    │
    ▼
P-Code指令序列（vector<PInstr>）
    │
    ├──▶ C++ VM（VM.h）：命令行解释执行
    │
    └──▶ JSON API → Node.js桥接 → 前端JS VM：可视化单步调试
```

**两层运行模式：**
- **普通模式**：`l26compiler.exe <file>` 编译并可选执行虚拟机
- **JSON模式**：`l26compiler.exe <file> --json` 输出结构化JSON供前端使用
- **前端模式**：`node server.js` 启动HTTP桥接，浏览器访问`localhost:8765`使用可视化界面

# 2 L26语言完整EBNF文法定义

## 2.1 EBNF元符号
- `::=` 定义；`|` 或；`{}` 0次及以上重复；`[]` 可选；双引号为终结符

## 2.2 完整EBNF文法（与PDF要求一致）

```ebnf
<program>        ::= <block>
<block>          ::= "{" <decls> <stmts> "}"
<decls>          ::= { <decl> }
<decl>           ::= <type> ID ";"
<type>           ::= "int" | "bool" | "set"

<stmts>          ::= { <stmt> }
<stmt>           ::= <assign_stmt> | <if_stmt> | <while_stmt>
                   | <io_stmt> | <block> | <set_op_stmt>

<assign_stmt>    ::= ID "=" <expr> ";"
<if_stmt>        ::= "if" "(" <bexpr> ")" <stmt> [ "else" <stmt> ]
<while_stmt>     ::= "while" "(" <bexpr> ")" <stmt>

<io_stmt>        ::= "write" <expr> ";" | "read" ID ";"
<set_op_stmt>    ::= "add" ID <aexpr> ";" | "remove" ID <aexpr> ";"

<expr>           ::= <bexpr> | <aexpr> | <set_expr>

/* 算术表达式 */
<aexpr>          ::= <aterm> { ("+" | "-") <aterm> }
<aterm>          ::= <afactor> { ("*" | "/") <afactor> }
<afactor>        ::= ID | NUM | "(" <aexpr> ")"

/* 逻辑表达式 */
<bexpr>          ::= <bterm> { "||" <bterm> }
<bterm>          ::= <bfactor> { "&&" <bfactor> }
<bfactor>        ::= "true" | "false" | "!" <bfactor>
                   | "(" <bexpr> ")" | <rel> | <set_test>

/* 关系运算 */
<rel>            ::= <aexpr> ("<" | "<=" | ">" | ">=" | "==" | "!=") <aexpr>

/* 集合表达式 */
<set_expr>       ::= "{" [ <aexpr> { "," <aexpr> } ] "}"
                   | ID "union" ID | ID "inter" ID

/* 集合测试 */
<set_test>       ::= <aexpr> "in" ID | "isempty" "(" ID ")"
```

## 2.3 关键字与运算符
- 关键字：int、bool、set、if、else、while、read、write、add、remove、union、inter、in、isempty、true、false
- 算术运算符：`+ - * /`
- 关系运算符：`< <= > >= == !=`
- 逻辑运算符：`! && ||`
- 集合运算符：`union inter in`

## 3.2 后端核心模块

### 符号表（SymTab.h）
栈式多级作用域，`vector<map<string,SymEntry>>`，支持：
- `EnterScope()`/`LeaveScope()`：进出块时压栈/弹栈
- `Declare(name, type)`：当前层声明，重复报错
- `Lookup(name)`：从栈顶向下逐层查找，实现变量遮蔽
- 变量偏移+3：前3单元为SL/DL/RA联系单元，对齐PL/0栈帧规范

### P-Code代码生成器（PCode.h）
- 指令结构体`PInstr{fct f, int l, int a}`，严格三字段格式
- 8种基础操作码：LIT/OPR/LOD/STO/CAL/INT/JMP/JPC
- 30+种OPR子码：标准PL/0运算 + 集合扩展（201-211）
- 跳转地址回填：`PatchJump()`解决if/while前向引用

### 虚拟机（VM.h）
- 标准PL/0栈式机：操作数栈`s[2048]`，寄存器p/b/t
- 静态链寻址：`Base(l, b)`沿s[b]向上l层
- 活动记录布局：SL(s[b+0]) + DL(s[b+1]) + RA(s[b+2]) + 局部变量
- 集合存储：`g_setPool[STACKSIZE]`，栈中仅存index，不破坏整型栈
- 集合200上限：`MAX_SET_SIZE=200`，SET_MK/SET_UNION/SET_ADD处检查

### 主程序（main.cpp）
- 普通模式：交互式编译执行，兼容ExpressionProject风格
- JSON模式：`--json`静默输出结构化JSON（tokens + pcode + errors），UTF-8编码中文错误信息

## 3.3 前端模块（Frontend/）

### server.js（Node.js桥接）
- 监听`localhost:8765`
- GET `/` → 提供index.html页面
- POST `/compile` → 接收源码，写临时文件，调`l26compiler.exe --json`，返回JSON
- 解决浏览器`file://`协议无法调用本地进程的跨域问题

### index.html（可视化前端）
- **JS VM**：与后端VM.h行为对齐的虚拟机，F码和OPR码完全一致
- **单步调试**：逐条执行P-Code，高亮当前PC，显示内存栈帧
- **编译调用**：通过fetch POST到后端桥接，获取tokens + pcode + errors
- **内存栈可视化**：沿静态链遍历所有帧，层级展示SL/DL/RA/局部变量，集合显示为`{1,2,3}`
- **错误展示**：单步至对应行显示编译错误，每个错误只显示一次

# 4 类P-Code规范与指令集设计

## 4.1 统一三元组格式
所有指令固定格式：`(地址) 助记符 l a`

## 4.2 八种基础指令

| 指令 | 格式 | 语义 |
|------|------|------|
| LIT | LIT 0 a | 常数a压栈 |
| OPR | OPR 0 x | 执行OPR子码x |
| LOD | LOD l a | 加载base(l,b)+a处变量压栈 |
| STO | STO l a | 弹出栈顶存入base(l,b)+a |
| CAL | CAL l a | 调用过程（L26暂不使用） |
| INT | INT 0 a | 栈顶指针上移a（a=局部变量数+3） |
| JMP | JMP 0 a | 无条件跳转 |
| JPC | JPC 0 a | 弹出栈顶，为0则跳转 |

## 4.3 OPR子码（与PL/0标准兼容 + 集合扩展）

**标准子码（0-20）：**
RET(0) / NEG(1) / ADD(2) / SUB(3) / MUL(4) / DIV(5) / ODD(6) /
EQ(8) / NE(9) / LT(10) / GE(11) / GT(12) / LE(13) /
WRITE(14) / WRITELN(15) / READ(16) /
AND(18) / OR(19) / NOT(20)

**集合扩展子码（201-211）：**
SET_MK(201)：弹cnt个整数构造集合（cnt存l字段）
SET_UNION(202)：弹两集合求并集
SET_INTER(203)：弹两集合求交集
SET_IN(204)：弹整数和集合测成员
SET_ISEMPTY(205)：测栈顶集合是否空
SET_ADD(206)：弹整数加入栈顶集合
SET_REMOVE(207)：弹整数从栈顶集合删除
SET_EQ(208)：测试两集合相等
WRITE_BOOL(209)：输出布尔值
WRITE_SET(210)：输出集合
SET_NE(211)：测试两集合不等

## 4.4 虚拟机栈帧布局

```
s[b+0] = SL（Static Link，外层基址）
s[b+1] = DL（Dynamic Link，调用方基址）
s[b+2] = RA（Return Address，返回地址）
s[b+3] = 局部变量0
s[b+4] = 局部变量1
...
```

Base(l,b)函数沿静态链向上l层寻址，与PL/0课件完全一致。

## 5.3 编译全流程

1. 词法分析：Scanner读取源文件，过滤空格/注释，输出Token流
2. 语法分析：Parser递归下降匹配EBNF，错误计数
3. 语义分析（同步执行）：符号表作用域管理、类型检查、集合运算合法性校验
4. 代码生成：调用PCodeGen发射P-Code，if/while先预留跳转后回填地址
5. 输出P-Code清单 / JSON输出
6. 虚拟机解释执行（可选）

## 5.4 跳转地址回填

if语句：解析条件 → `JPC 0 -1`(暂占) → 解析then体 → `JMP 0 -1`(暂占) → `PatchJump(labelElse)` → [解析else体] → `PatchJump(labelEnd)`

while语句：记录labelTop → 解析条件 → `JPC 0 labelEnd`(暂占) → 解析循环体 → `JMP 0 labelTop` → `PatchJump(labelEnd)`

# 6 测试用例与运行结果

## 6.1 测试用例1（PDF示例1）：read + in + add + while
```l26
{
    int val;
    set s;
    s = {1, 2, 3};
    read val;
    if (val in s) {
        add s 10;
        write val;
    }
    while (val > 0) {
        val = val - 1;
        write val;
    }
    write 999;
}
```
**覆盖功能**：read输入、in成员判断、add集合添加、while循环、write输出

## 6.2 测试用例2（PDF示例2）：嵌套作用域 + 变量遮蔽
```l26
{
    int x;
    set s;
    x = 10;
    s = {1, 2};
    {
        set x;
        x = {5, 6};
        add x 7;
        write x;
        write s;
    }
    write x;
}
```
**覆盖功能**：嵌套块、变量遮蔽（内层set x遮蔽外层int x）、集合字面量、add、write set

## 6.3 测试用例3：集合运算 + bool表达式 + remove + isempty
```l26
{
    int a; int b;
    set s1; set s2; set s3; bool flag;
    a = 5; b = 3;
    s1 = {1, 2, 3, 4, 5};
    s2 = {3, 4, 5, 6, 7};
    s3 = s1 union s2; write s3;
    s3 = s1 inter s2; write s3;
    remove s1 3; write s1;
    flag = isempty(s1); write flag;
    if ((a > b) && (a != 0)) { write a; }
}
```
**覆盖功能**：union/inter/remove/isempty/布尔表达式/关系运算

## 6.4 集合相等测试（加分项）
```l26
{
    set a; set b; bool r;
    a = {1, 2, 3}; b = {1, 2, 3};
    r = (a == b);       // SET_EQ → true
    b = {4, 5};
    r = (a != b);       // SET_NE → true
    write r;
}
```
**覆盖功能**：集合相等/不等判定

## 7.2 单步调试功能

- 开始单步：编译源码，初始化VM，准备逐条执行
