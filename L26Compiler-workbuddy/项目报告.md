# 1 项目概述
## 1.1 作业需求说明
根据《2026编译原理大作业要求》：
1. 实现扩展L26语言编译器，支持`int`整型、`bool`布尔、`set`集合三种数据类型；
2. 编译器输出标准类P-Code三字段指令，配套虚拟机解释运行；
3. 语言语法包含块、变量声明、赋值、if-else、while循环、读写语句、集合专用语法；
4. 集合功能：集合字面量`{}`、`union`并集、`inter`交集、`add/remove`增删、`in`成员判断、`isempty`空集判断、集合相等`==/!=`；
5. 支持多层嵌套作用域、变量遮蔽、完整静态语义检查（未定义变量、类型不匹配、非法集合运算报错）；
6. 输出P-Code汇编清单，可选择虚拟机执行目标代码；
7. 交付完整项目代码、多组测试用例、标准化设计文档。

## 1.2 开发方案选择
本项目选择**第三组 Coco/R C++实现**：
- 优势：Coco/R自动生成词法扫描器Scanner、语法解析器Parser，专注语义分析与代码生成；
- 对比手工C、Flex/Bison：开发效率更高，递归下降自动生成，符合课程PL0递归下降教学思路；
- 语言扩展：在标准PL0基础上新增bool、set全套语义与P-Code扩展指令。

## 1.3 整体架构总览
采用**单遍递归下降编译器 + 独立解释器**架构，分为四层：
1. 词法层：Coco/R自动生成Scanner，识别标识符、数字、关键字、运算符；
2. 语法语义层：Parser结合自定义符号表做作用域管理、类型校验、语义报错；
3. 中间代码层：PCodeGen生成标准三字段类P-Code指令，支持跳转地址回填；
4. 运行层：VM虚拟机加载P-Code代码，基于栈式机模型解释执行，支持整数、布尔、集合运行时存储。

**整体数据流：**`.l26`源文件 → Scanner单词流 → Parser语法树遍历 → 语义检查 + P-Code生成 → 输出指令清单 → VM加载执行并打印输出。

# 2 L26扩展语言完整EBNF文法定义
## 2.1 EBNF元符号
- `::=` 定义；`|` 或；`{}` 0次及以上重复；`[]` 可选；`()` 分组；双引号为终结符；
- 非终结符：<program>、<block>、<decl>等；
- 标识符ID：字母开头，字母数字下划线组合；NUM：十进制整数。

## 2.2 完整扩展EBNF文法（含set集合）
```ebnf
<program>        ::= <block>
<block>          ::= "{" <decls> <stmts> "}"
<decls>          ::= { <decl> }
<decl>           ::= <type> ID ";"
<type>           ::= "int" | "bool" | "set"

<stmts>          ::= { <stmt> }
<stmt>           ::= <assign_stmt> | <if_stmt> | <while_stmt> | <io_stmt> | <block> | <set_op_stmt>

<assign_stmt>    ::= ID "=" <expr> ";"
<if_stmt>        ::= "if" "(" <bexpr> ")" <stmt> [ "else" <stmt> ]
<while_stmt>     ::= "while" "(" <bexpr> ")" <stmt>

<io_stmt>        ::= "write" <expr> ";" | "read" ID ";"
<set_op_stmt>    ::= "add" ID <aexpr> ";" | "remove" ID <aexpr> ";"

<expr>           ::= <bexpr> | <aexpr> | <set_expr>

/* 算术表达式 int */
<aexpr>          ::= <aterm> { ("+" | "-") <aterm> }
<aterm>          ::= <afactor> { ("*" | "/") <afactor> }
<afactor>        ::= ID | NUM | "-" <afactor> | "(" <aexpr> ")"

/* 逻辑表达式 */
<bexpr>          ::= <bterm> { "||" <bterm> }
<bterm>          ::= <bfactor> { "&&" <bfactor> }
<bfactor>        ::= "true" | "false" | "!" <bfactor> | "(" <bexpr>
                   | <rel> | <set_test>

/* 关系运算 */
<rel>            ::= <aexpr> ("<" | "<=" | ">" | ">=" | "==" | "!=") <aexpr>

/* 集合表达式 set */
<set_expr>       ::= "{" [ <aexpr> { "," <aexpr> } ] "}"
                   | ID ( "union" ID | "inter" ID )

/* 集合测试 */
<set_test>       ::= <aexpr> "in" ID | "isempty" "(" ID ")"
```

## 2.3 关键字与运算符
### 关键字
int、bool、set、if、else、while、read、write、add、remove、union、inter、in、isempty、true、false
### 运算符
算术：`+ - * /`
关系：`< <= > >= == !=`
逻辑：`! && ||`
集合专用：`union inter in`

# 3 项目代码分层结构设计
## 3.1 全部源文件总览
| 文件名 | 模块功能 |
|--------|----------|
| L26.atg | Coco/R文法源文件，定义语法+语义动作，编译生成Scanner.h/cpp、Parser.h/cpp |

## 3.2 符号表模块 SymTab.h
### 核心设计目标
实现**栈式多级作用域**，匹配块`{}`的进入/退出，支持变量遮蔽，记录变量：名称、类型、嵌套层次、局部偏移。
### 核心结构体 SymEntry
```cpp
struct SymEntry {
    std::string name;     // 变量名
    std::string typName;  // int/bool/set
    int level;            // 定义所在嵌套层
    int offset;           // 块内局部偏移
};
```
### SymTab类核心方法
1. `EnterScope()`：进入`{`，新建一层作用域，偏移初始化为0；
2. `LeaveScope()`：退出`}`，销毁当前最内层作用域；
3. `Decl()`：声明变量，检测同层重定义，分配偏移；
4. `Lookup()`：从当前层向上遍历符号表，查找变量，实现内层遮蔽外层；
5. `CurrentOffset()`：获取当前块下一个可用局部偏移。
### 存储规则
变量偏移+3：前3个单元固定为活动记录SL静态链、DL动态链、RA返回地址，变量从偏移3开始存储，完全匹配PL0栈帧规范。

## 3.3 P-Code代码生成模块 PCode.h
### 1）单条指令结构 PInstr
严格遵循PPT三元组`f l a`：
```cpp
struct PInstr {
    fct f;  // 操作码
    int l;  // 层次差
    int a;  // 地址/常量/OP子码
};
```
### 2）基础操作码枚举 fct
完全复刻PL0标准指令，添加F_前缀规避Windows宏冲突：
`F_LIT F_OPR F_LOD F_STO F_CAL F_INT F_JMP F_JPC`
### 3）OPR运算子码 OprCode
1. 保留标准PL0：加减乘除、正负、奇偶、关系运算、读写、返回；
2. 扩展集合专属子码：集合构造、交并、增删、成员判断、集合相等、集合打印；
### 4）代码生成器 PCodeGen
核心功能：
- `Emit()`：底层发射任意三字段指令；
- 封装快捷函数：`EmitLIT/EmitLOD/EmitSTO/EmitJPC`等；
- `PatchJump()`：跳转地址回填，解决if/while未知跳转地址；
- `PrintListing()`：格式化打印全部P-Code汇编清单；
- 集合专用发射函数：`EmitSetMake/EmitSetUnion/EmitSetAdd`等。

## 3.4 虚拟机解释器模块 VM.h
### 栈式机核心寄存器（与PL0 PPT完全一致）
- `p`：程序计数器，当前待执行指令下标；
- `b`：基址寄存器，当前活动记录起始地址；
- `t`：栈顶寄存器；
- `s[]`：全局整数运行栈；
### 集合扩展存储
`std::set<int> g_setPool[]` 全局集合池，栈中仅存储集合池下标，不改动原有整数栈逻辑；
### 核心函数
1. `Base(int l, int cur_b)`：沿静态链向上l层，计算跨层变量基地址（PPT标准寻址算法）；
2. `Run()`：完整从头执行P-Code；
3. `Step()`：单步执行（加分项功能）；
4. `ExecInstr()`：分发8类基础指令；
5. `ExecOPR()`：处理算术、逻辑、IO、全部集合运算；
### 指令执行逻辑完全对齐PL0教学案例：
- CAL调用：生成SL/DL/RA三链单元；
- INT开辟栈帧，a=局部变量+3；
- OPR_RET销毁栈帧，恢复b、p、t；
- LOD/STO通过Base实现多层变量访问。

执行流程：

4. 检测编译错误，有错直接退出；
5. 尾部自动发射RET返回指令，打印完整P-Code清单；
6. 交互询问是否运行虚拟机；

# 4 类P-Code规范与PL0 PPT对照
## 4.1 统一三元组指令格式
所有指令固定 `(地址) 助记符 l a`，与课件PL0完全一致：
示例：`(11) LOD 0 3`
- f=LOD：取变量；l=0层次差；a=3变量偏移

## 4.2 核心基础指令与标准完全一致
| 指令 | PPT功能 | 本项目实现 |
|------|---------|------------|
| LIT 0 a | 常量a压栈 | 完全一致 EmitLIT |
| LOD l a | 层差l，偏移a取变量入栈 | Base寻址完全相同 |
| STO l a | 栈顶存入层差l偏移a变量 | 逻辑无修改 |
| INT 0 n | 开辟n个栈单元，n=局部变量+3 | 文法自动计算declCnt+3 |
| CAL l a | 调用地址a，层差l | 栈SL/DL/RA创建逻辑和案例相同 |
| OPR 0 0 | 过程返回RET | 栈帧销毁逻辑完全复刻 |
| JMP 0 a | 无条件跳转a | 标准实现 |
| JPC 0 a | 栈顶为0跳转，栈顶-1 | 标准实现 |

## 4.3 集合扩展OPR子码说明
基础PL0指令无任何修改，仅在OPR运算码中新增集合专用子码，属于作业语言扩展允许内容，不破坏标准P-Code体系：
集合构造、交、并、增、删、成员判断、空集、集合相等、集合打印。

## 4.4 虚拟机栈帧与静态链寻址
1. 每个块/过程活动记录前3单元固定：
`s[b] = SL静态链；s[b+1]=DL动态链；s[b+2]=RA返回地址`
2. `Base(l)`循环l次取`s[b]`向上跳转外层基址，课件标准跨层变量寻址算法；
3. 变量偏移从3开始，避免覆盖三链单元，和PPT案例内存布局一致。

# 5 编译器完整运行流程
## 5.1 编译全流程
1. **词法分析**：Scanner读取源文件，过滤空格换行注释，输出单词流（标识符、数字、关键字）；
2. **语法分析**：Parser递归下降匹配EBNF文法，遇到语法错误计数；
3. **语义分析（同步执行）**
   - 块进入/退出管理作用域；
   - 变量声明查重、分配偏移；
   - 变量查找，检测未定义；
   - 表达式类型推导，赋值/运算类型匹配校验；
   - 集合运算合法性校验；
4. **中间代码生成**：遍历语法分支，调用PCodeGen发射对应P-Code指令，if/while先预留跳转，解析完分支后回填地址；
5. **编译产物输出**：控制台打印全部P-Code指令清单；
6. **虚拟机解释（可选）**：加载代码，逐指令执行，控制台输出读写结果、集合打印内容。

## 5.2 源文件编译执行流程
1. 编写`.l26`测试源码；
2. 编译Coco/R文法生成Scanner、Parser；
3. 整体编译项目生成exe可执行文件；
4. 运行exe，输入源码文件路径；
5. 无语法语义错误后打印P-Code；
6. 输入y启动虚拟机，查看运行输出。

## 5.3 跳转地址回填机制
if/while处理逻辑：
1. 解析布尔条件表达式，生成条件跳转`JPC 0 -1`（临时占位地址）；
2. 保存该跳转指令下标；
3. 解析分支内部语句；
4. 使用`PatchJump(下标)`，将指令a字段修改为当前代码长度（真实跳转地址）；
完全匹配课件PL0跳转回填实现方式。
