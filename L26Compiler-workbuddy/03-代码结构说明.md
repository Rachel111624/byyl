## 2. 各文件职责

| 文件 | 职责 | 来源 |
|------|------|------|
| `L26.atg` | Coco/R 文法源文件，定义 L26 全部语法规则 + 嵌入语义动作（符号表操作、P-Code 生成、类型感知的集合相等/不等判断） | 手工编写【核心】 |
| `SymTab.h` | 符号表实现：栈式多级作用域，支持 `EnterScope/LeaveScope/Declare/Lookup` | 手工编写 |
| `PCode.h` | 标准 PL/0 三字段 P-Code 指令（`fct f, int l, int a`）、`PInstr` 结构 + `PCodeGen` 代码生成器（含 `EmitLIT/EmitLOD/EmitSTO/EmitINT/EmitJMP/EmitJPC/EmitOPR/EmitSetEQ/EmitSetNE` 等方法 + `PrintListing()`） | 手工编写 |
| `VM.h` | 类 P-Code 虚拟机：操作数栈 + 帧栈 + `g_setPool` 集合存储，支持全部 L26 运行时操作（算术/逻辑/集合/I/O/跳转） | 手工编写 |
| `main.cpp` | 程序入口：支持命令行参数 `./l26compiler.exe <file>` 或交互输入；读取文件 → Scanner → Parser → P-Code → VM；风格 100% 对标 ExpressionProject | 手工编写 |
| `Parser.frame` | Coco/R 生成模板（修改了 `-->headerdef` 注入了 `SymTab.h` 和 `PCode.h` 的 include） | 手工修改 |
| `Scanner.cpp/h` | 词法分析器（Coco/R 自动生成，不手动修改） | **coco.exe 生成** |
| `Parser.cpp/h` | 语法分析器（Coco/R 自动生成，内含 ATG 语义动作编译结果 + 谓词函数） | **coco.exe 生成** |
| `Frontend/index.html` | 自包含可视化页面：JS 词法分析 + 递归下降编译 + JS 虚拟机，支持单步调试 | 手工编写 |
| `Tests/test*.l26` | L26 测试程序（3 个） | 手工编写 |
| `Makefile` | MinGW g++ 构建脚本，对标 ExpressionProject tasks.txt 的构建命令 | 手工编写 |

## 3. Windows 环境完整运行步骤

### 3.1 环境准备

| 工具 | 版本要求 | 下载地址 / 说明 |
|------|----------|-----------------|
| **Coco/R for C++** | 任意版本（coco.exe） | F:\CocoCPP\bin\Coco.exe |
| **MinGW-w64 g++** | ≥ 5.0，支持 C++11 | https://www.mingw-w64.org/ 或 MSYS2 |
| **coco.exe** 已在 PATH 或使用绝对路径 | 能在命令行直接调用 | 例如：`F:\CocoCPP\bin\Coco.exe` |

验证安装：
```bat
g++ --version      :: 查看 g++ 版本
```

### 3.2 生成词法/语法分析器（Coco/R 步骤）

```bat
cd F:\L26Compiler-workbuddy\Compiler

:: 用 Coco/R 从 ATG 文件生成 Scanner / Parser
coco.exe L26.atg -namespace L26Project
```

执行后，当前目录应新增（或更新）：
- `Scanner.cpp`、`Scanner.h`
- `Parser.cpp`、`Parser.h`

> **说明**：`-namespace L26Project` 使生成代码与 `SymTab.h`、`PCode.h`、`VM.h`、`main.cpp` 的命名空间一致，对标 ExpressionProject 的命名空间做法。

### 3.3 编译 C++ 源码（MinGW g++）

```bat
:: 方式1：直接编译（只需 3 个 cpp 文件）
g++ -g -std=c++11 -Wall Scanner.cpp Parser.cpp main.cpp -o l26compiler.exe

:: 方式2：使用 Makefile（一键生成 + 编译）
make build
```

### 3.4 运行编译器（支持命令行参数）

```bat
:: 方式1：命令行参数（推荐）
l26compiler.exe test_full.l26

:: 方式2：交互输入
l26compiler.exe
请输入 L26 源文件路径 (例如: test1.l26): test_full.l26

:: 方式3：批处理
echo n | l26compiler.exe test_full.l26    （自动回答 n 跳过 VM 执行）
```

程序流程：
1. 词法/语法/语义分析 → 通过则显示 `>>> 语法/语义检查通过！`
2. 输出 P-Code 清单
3. 提示 `是否执行虚拟机？(y/n):` — 输入 y 执行，n 跳过

**当前验证通过的测试用例**：

| 测试文件 | 覆盖功能 |
|----------|----------|
| `test1.l26` | 算术 + bool 字面量 + 关系表达式 + 集合字面量 |
| `test_bool2.l26` | true/false 赋值 |
| `test_rel.l26` | x > 0 关系表达式 |
| `test_simple.l26` | 算术表达式 |
| `test_full.l26` | 全集：算术/布尔/集合/`in`/`isempty`/`union`/`inter`/`if`/`while`/`write` |
| `test_seteq.l26` | 集合相等（`==` → SET_EQ）/ 集合不等（`!=` → SET_NE） |

### 3.5 使用 Makefile 一键构建

```bat
:: 在 VSCode 终端（CMD 或 PowerShell）或 Git Bash 中
cd F:\L26Compiler-workbuddy\Compiler
make build       :: coco.exe 生成 + g++ 编译
make clean       :: 清除生成文件
```

## 4. 前端页面运行方式

无需服务器，直接双击打开：

```
F:\L26Compiler-workbuddy\Frontend\index.html
```

或在 VSCode 中右键 `index.html` → **Open with Live Server**

操作流程：
1. 在左侧「源代码」编辑框输入 L26 程序（或点击「示例1/2/3」加载示例）
2. 点击「▶ 编译 + 执行」：自动词法分析 → 显示 Token 列表 → 生成 P-Code → 执行 VM → 显示输出
3. 遇到 `read` 语句时弹出输入对话框，输入整数后确认
4. 点击「⏭ 开始单步」后，每次点击「→ 单步执行」执行一条 P-Code 指令，右下角实时显示内存帧状态

**按钮全部生效的保证**：前端 `index.html` 中 `stepOnce()` 函数已修复了缺失的 `async function` 声明，grid 布局也修正为 3 行结构。

## 5. 对标 ExpressionProject 的编码规范说明

| 规范项 | ExpressionProject 做法 | L26Compiler 对标 |
|--------|----------------------|-----------------|
| 命名空间 | `UserProject` | `L26Project` |
| 命名空间注入 | ATG `$namespace=UserProject` | ATG `$namespace=L26Project` |
| main.cpp 风格 | `system("chcp 65001")` + 交互输入 / 命令行参数 | 完全一致，新增命令行参数支持 |
| wchar_t 路径 | `coco_string_create(path)` | 完全一致 |
| 头文件保护 | `#ifndef COCO_*_H__` | `#ifndef L26_*_H__` |
| 错误输出 | Coco/R 默认机制 | Coco/R 默认机制（继承） |
| 文件拆分 | Scanner + Parser + main.cpp | 新增 SymTab.h / PCode.h / VM.h / Parser.frame（手工模块）+ 生成文件 |
| 注释风格 | `/* ... */` + `//` | 完全一致 |
| 缩进 | 4 空格 | 完全一致 |
| 构建方式 | g++ 直接编译 / Makefile | 完全对标 |
